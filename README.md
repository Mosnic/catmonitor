# Cat Monitor

An autonomous, AI-powered monitoring system for stray cat feeding stations. This application uses Vision-Language Models (VLMs) and vector embeddings to identify individual cats, track their health, and analyze visit patterns over time.

## 🌟 Features

* **Real-time Video Processing**: Automatically detects new clips from security cameras (via FTP) and triggers an agentic analysis loop.
* **Intelligent Identification**: Uses `ChromaDB` and image embeddings to recognize returning cats while flagging "uncertain" matches for later review.
* **Reflective Reasoning**: A periodic "Reflective Agent" reviews historical data to detect health declines, identify long-term patterns, and resolve identity links.
* **Multi-Modal Analysis**:
    * **Vision**: Extracts and scores the best frames from video for clear identification.
    * **Weather**: Automatically correlates visits with local weather conditions (temperature, precipitation, wind) at the time of recording.
* **Statistical Insights**: Capable of generating and running custom Python code to perform complex statistical analysis on visit frequency and behavior.

## 🏗️ Architecture

The system consists of two primary loops:

1.  **Reactive Loop (`watcher.py`)**: A filesystem observer that monitors an FTP drop folder. When a video arrives, it extracts the best frames and performs an initial "agentic" pass to log the visit.
2.  **Reflective Loop (`reflective_loop.py`)**: A background process that "thinks" about the data. It looks for trends, manages the cat database, and flags concerns like extended absences or health changes.

## 🛠️ Tech Stack

* **Language**: Python 3.10+
* **AI Models**: Qwen3-VL-8B-Instruct (via vLLM) for vision and reasoning.
* **Database**: SQLite for structured records and ChromaDB for vector embeddings.
* **Tools**: FFmpeg for frame extraction and Open-Meteo for weather data.

## 🚀 Getting Started

### Prerequisites

* A running **vLLM** server hosting a supported Vision-Language Model.
* **FFmpeg** installed on your system path.
* Python 3.10 or higher.

### Installation

1.  **Clone the repository**:
    ```bash
    git clone https://github.com/yourusername/catmonitor.git
    cd catmonitor
    ```

2.  **Install dependencies**:
    ```bash
    pip install -r requirements.txt
    ```

3.  **Configure Environment Variables**:
    ```bash
    export VLLM_URL="http://your-vllm-server:8000/v1/chat/completions"
    export VLLM_MODEL="Qwen/Qwen3-VL-8B-Instruct"
    export FTP_FOLDER="/path/to/your/camera/uploads"
    ```

4.  **Run the Monitor**:
    ```bash
    python watcher.py
    ```

## 📊 Database Schema

The system maintains several key tables in its SQLite database:
* **`cats`**: Core profiles, including descriptions and first/last seen timestamps.
* **`visits`**: Detailed logs of every appearance, including weather, behavior, and health flags.
* **`alerts`**: High-priority findings generated by the reflective agent (e.g., `health_decline`, `extended_absence`).
* **`links`**: Provisional identity matches awaiting human or agent confirmation.
